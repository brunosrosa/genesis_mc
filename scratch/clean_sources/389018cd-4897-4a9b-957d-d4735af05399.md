OpenClaw Settings Configuration centralizes all runtime and installation-specific settings in a single JSON file at ~/.openclaw/openclaw.json. It provides a structured, extensible schema for agent routing and metadata, model assignments (primary and fallback lists), provider authentication, skill-specific entries, cron job scheduling, gateway and plugin settings, and subagent definitions. Use it to define agent names/roles, enforce system-wide defaults (e.g., agents.defaults.model and agents.defaults.maxConcurrent — note per-agent maxConcurrent is not supported), enable or customize skills via skills.entries, and control model/provider fallbacks for resilience. Key advantages include a single source of truth for a 16-agent deployment, per-installation customization, persistence across updates, and straightforward automation and troubleshooting for developer and DevOps workflows.
