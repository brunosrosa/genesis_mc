When an AI agent performs an action - whether responding to a user, executing a terminal command, or interacting with a GUI - the subsequent response (next-state signal) contains rich feedback about the quality and appropriateness of that action. Current systems typically treat these signals merely as context for the next action, discarding their potential as real-time learning sources.\n\nThe authors identify two types of underutilized signals: evaluative signals that implicitly score an action's quality (like user re-queries indicating dissatisfaction), and directive signals that provide explicit guidance on how actions should be improved (such as user corrections or detailed error messages). This represents a significant missed opportunity, as these signals are universally generated across all agent interactions and could enable continuous, online learning from live deployments.\n\nTechnical Framework and Architecture\n\nOpenClaw-RL implements an asynchronous, fully decoupled architecture built on four independent loops that operate simultaneously without blocking each other:\n\nPolicy Serving\n\n: Handles incoming agent requests using SGLang\n\nEnvironment Hosting\n\n: Manages interactions through HTTP/API endpoints\n\nPRM Judging\n\n: Evaluates rewards from next-state signals\n\nPolicy Training\n\n: Updates agent policies using Megatron\n\nThis decoupling ensures zero interruption to live agent serving while enabling continuous learning from real-time interaction streams.
