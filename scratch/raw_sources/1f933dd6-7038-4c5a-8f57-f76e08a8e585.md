Skills Ecosystem Overview\n\n1.1 What Is an OpenClaw Skill\n\nIn the OpenClaw architecture, a Skill is a self-contained capability module that tells the agent \"what I can do\" and \"what resources I need to do it\" through a standardized interface. Unlike traditional plugin systems that require the host program to mount specific APIs, OpenClaw Skills use a declarative design: developers describe skill capabilities in human-readable format within the \n\nskill.md\n\n file, and the OpenClaw core engine dynamically reads these descriptions when planning tasks to determine when to invoke which skill. \n\n[2]\n\nhttps://www.meta-intelligence.tech/en/insight-openclaw-skills#ref-2\n\nThis design brings two important advantages: first, the agent can understand each skill's capability boundaries before execution, avoiding trial-and-error invocations; second, humans can directly read the \n\nskill.md\n\n to audit skill behavior without needing to reverse-engineer code. This is especially important for enterprise compliance reviews.\n\nEach Skill is represented as an independent directory in the file system, structured as follows:\n\nmy-skill/\n├── skill.md          # Skill specification (required)\n├── index.js          # Main logic entry point (required)\n├── package.json      # npm dependency declaration (optional)\n├── tests/            # Unit test directory (recommended)\n│   └── skill.test.js\n└── README.md         # Additional documentation (optional)Copy\n\n\n1.2 Official Skills Marketplace Status\n\nAs of February 2026, the OpenClaw official Skills Marketplace has cataloged over 100 skills, contributed by the official core team, verified partners, and community developers. \n\n[3]\n\nhttps://www.meta-intelligence.tech/en/insight-openclaw-skills#ref-3\n\n The distribution by functional category is as follows:\n\nBrowser Automation\n\n: Approximately 22, including web scraping, form filling, screenshots, multi-tab management, and more\n\nFile System Operations\n\n: Approximately 15, covering read/write, search, compression, and format conversion\n\nMessaging & Communication\n\n: Approximately 18, integrating Slack, Discord, Email, Telegram, and other platforms; Both are indispensable: YAML determines the skill's discoverability and execution boundaries, while Markdown determines whether the agent can correctly understand when and how to use the skill. \n\n[2]\n\nhttps://www.meta-intelligence.tech/en/insight-openclaw-skills#ref-2\n\nBelow is a complete \n\nskill.md\n\n structure example:\n\n---\nname: my-custom-skill\nversion: 1.2.0\ndescription: \"A one-line description of this skill's core functionality\"\nauthor: your-username\nlicense: MIT\n\ncapabilities:\n  - id: fetch-webpage\n    description: \"Fetch the content of a specified URL\"\n  - id: extract-text\n    description: \"Extract plain text paragraphs from HTML\"\n\npermissions:\n  network: true\n  filesystem: false\n  shell: false\n  env:\n    - MY_API_KEY\n\ninputs:\n  - name: url\n    type: string\n    required: true\n    description: \"The full URL of the target webpage\"\n  - name: selector\n    type: string\n    required: false\n    default: \"body\"\n    description: \"CSS selector to specify the extraction scope\"\n\noutputs:\n  - name: text\n    type: string\n    description: \"The extracted plain text content\"\n  - name: wordCount\n    type: number\n    description: \"Word count statistic\"\n\ntags:\n  - browser\n  - scraping\n  - text-extraction\n\nminOpenClawVersion: \"2.1.0\"\n---\n\n## Skill Description\n\nThis skill is used to extract text content from specified web pages,\nsuitable for scenarios requiring quick access to web information,\ncontent analysis, or building data pipelines.\n\n## Usage Examples\n\nPlease extract the main body text from https://example.com and count the words.\n\n## Notes\n\n- Does not support pages requiring login authentication\n- JavaScript dynamically rendered content may not be fully captured\n- Please comply with the target website's robots.txt rulesCopy\n\n\n2.2 Required Fields Explained\n\nname\n\n : The unique identifier for the skill, using kebab-case format (e.g., \n\nblog-watcher\n\n ). Names must not be duplicated within the same OpenClaw instance. When published to the marketplace, names are prefixed with the author, e.g., \n\n@username/blog-watcher\n\n .\n\nversion\n\n : Follows the Semantic Versioning specification, formatted as \n\nMAJOR.MINOR.PATCH\n\n . Major version changes indicate breaking API changes, minor versions indicate new features, and patch versions indicate bug fixes.\n\ncapabilities\n\n : This is the most critical field in \n\nskill.md\n\n , directly determining whether the OpenClaw agent can invoke this skill at the appropriate time. Each capability requires an \n\nid\n\n (machine-readable identifier) and a \n\ndescription\n\n (natural language description for AI reasoning). Descriptions should be as specific as possible, avoiding vague terms like \"process\" or \"manage,\" and instead using verb-plus-object structures such as \"Send a Slack message to a specified channel.\"\n\npermissions\n\n : Declares the system resource access permissions required for skill execution. OpenClaw follows the principle of least privilege, and only permissions declared here can be executed. The main permission types include:\n\nnetwork: true/false\n\n : Whether external network requests can be made\n\nfilesystem: true/false\n\n : Whether the local file system can be read/written\n\nshell: true/false\n\n : Whether system commands can be executed (high risk, requires special review)\n\nenv: [...]\n\n : List of environment variable names allowed to be read\n\nclipboard: true/false\n\n : Whether the clipboard can be accessed\n\n2.3 Input/Output Specification\n\nThe \n\ninputs\n\n field defines the parameters a skill accepts, with each parameter containing: \n\nname\n\n (parameter name), \n\ntype\n\n (data type: \n\nstring\n\n , \n\nnumber\n\n , \n\nboolean\n\n , \n\narray\n\n , \n\nobject\n\n ), \n\nrequired\n\n (whether mandatory), \n\ndefault\n\n (default value), and \n\ndescription\n\n (explanation). Clear inputs definitions enable the agent to automatically extract parameters from conversations, reducing the number of times users need to explicitly specify parameters.\n\nThe \n\noutputs\n\n field describes the data structure returned by the skill, helping the agent correctly pass this skill's output to the next step when planning multi-step tasks.\n\n2.4 Version Compatibility Management\n\nThe \n\nminOpenClawVersion\n\n field specifies the minimum OpenClaw version required by the skill, which is especially important for skills that use newer API features. If the user's OpenClaw version does not meet the requirement, the system will display a warning before installation, rather than letting the skill silently fail at runtime.\n\n3.
