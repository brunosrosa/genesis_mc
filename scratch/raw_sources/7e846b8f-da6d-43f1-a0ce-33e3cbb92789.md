(formerly Clawdbot & Moltbot) is a \n\nlocal-first\n\n AI agent that runs on your machine. It combines multiple capabilities into a single assistant: it remembers conversations, adapts to your usage, runs continuously, uses context from your files and apps, and can be extended with community \n\nskills\n\n.\n\nRunning OpenClaw and its LLMs \n\nfully on your DGX Spark\n\n keeps your data private and avoids ongoing cloud API costs. DGX Spark is well suited for this: it runs Linux, is designed to stay on, and has \n\n128GB memory\n\n, so you can run large local models for better accuracy and more capable behavior.\n\nWhat you'll accomplish\n\nYou will have OpenClaw installed on your DGX Spark and connected to a local LLM (via LM Studio or Ollama). You can use the OpenClaw web UI to chat with your agent, and optionally connect communication channels and skills. The agent and models run entirely on your Spark—no data leaves your machine unless you add cloud or external integrations.\n\nPopular use cases\n\nPersonal secretary\n\n: With access to your inbox, calendar, and files, OpenClaw can help manage your schedule, draft replies, send reminders, and find meeting slots.\n\nProactive project management\n\n: Check project status over email or messaging, send status updates, and follow up or send reminders.\n\nResearch agent\n\n: Combine web search and your local files to produce reports with personalized context.\n\nInstall helper\n\n: Search for apps/libraries, run installations, and debug errors using terminal access (larger models recommended).\n\nWhat to know before starting\n\nBasic use of the Linux terminal and a text editor\n\nOptional: familiarity with Ollama or LM Studio if you plan to use a local model\n\nAwareness of the security considerations below\n\nImportant: security and risks\n\nAI agents can introduce real risks. Read OpenClaw's guidance: \n\nOpenClaw Gateway Security\n\nhttps://docs.openclaw.ai/gateway/security\n\n.\n\nMain risks:\n\nData exposure\n\n: Personal information or files may be leaked or stolen.\n\nMalicious code\n\n: The agent or connected tools may expose you to malware or attacks.\n\nYou cannot eliminate all risk; proceed at your own risk. \n\nCritical security measures:\n\nSTRONGLY RECOMMENDED:\n\n Run OpenClaw on a dedicated or isolated system (e.g., a clean DGX Spark or VM) and only copy in the data the agent needs. Do not run this on your primary workstation with sensitive data.\n\nUse \n\ndedicated accounts\n\n for the agent instead of your main accounts; grant only the minimum access it needs.\n\nEnable only \n\nskills you trust\n\n, preferably those vetted by the community. Skills that provide terminal or file system access increase risk significantly.\n\nCRITICAL:\n\n Ensure the OpenClaw web UI and any messaging channels are \n\nnever exposed\n\n to the public internet without strong authentication. Use SSH tunneling or VPN if accessing remotely.\n\nWhere possible, \n\nlimit internet access\n\n for the agent using firewall rules or network isolation.\n\nMonitor activity\n\n: Regularly review logs and commands executed by the agent.\n\nPrerequisites\n\nDGX Spark running Linux, connected to your network\n\nTerminal (SSH or local) access to the Spark\n\nFor local LLMs: enough GPU memory for your chosen model (see Instructions for size guidance; DGX Spark's 128GB supports large models)\n\nTime and risk\n\nDuration\n\n: About 30 minutes for install and first-time model setup; model download time depends on size and network (gpt-oss-120b is ~65GB and may take longer on slower connections).\n\nRisk level\n\n: \n\nMedium to High\n\n—the agent has access to whatever files, tools, and channels you configure.
